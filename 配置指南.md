# Codex + cc-switch 接入 DeepSeek 配置指南

## 架构

```
Codex CLI / 桌面端  →  cc-switch (路由)  →  本地适配器 (127.0.0.1:9468)  →  DeepSeek API
  Responses API           供应商切换            Responses ↔ Chat 转换           Chat Completions
```

DeepSeek 只提供 Chat Completions API，Codex 用的是 Responses API，所以需要本地适配器做协议转换。

## 已完成的配置

- 适配器目录：`C:\Users\Administrator\dsv4-local-adapter\`
- 监听地址：`http://127.0.0.1:9468/v1`
- 环境变量已设置：`UPSTREAM_BASE_URL`、`UPSTREAM_CHAT_PATH`、`UPSTREAM_MODEL`

## 使用步骤

### 1. 启动适配器

双击 `C:\Users\Administrator\dsv4-local-adapter\启动中转工具.bat`

### 2. 填入 API Key

在 GUI 中：
- **上游 API Key** 栏填入你的 DeepSeek API Key
- 模型默认 `deepseek-v4-pro`，也可切换为 `deepseek-v4-flash`
- 点击 **保存配置**
- 点击 **测试上游直连** 确认连通

### 3. 导入到 cc-switch

点击 GUI 中的 **导入到 CC** → cc-switch 弹出确认窗口 → 确认导入。

然后在 cc-switch 中将 Codex 的当前供应商切换为 **"DeepSeek V4 本地中转"**。

### 4. 应用 Codex 配置

点击 GUI 中的 **应用 Codex 配置**，自动写入 `~/.codex/config.toml`。

### 5. 验证

```powershell
codex "hello, say hi"
```

正常回复即表示链路通。Codex 桌面端同样生效。

## 后台模式（不用 GUI）

```powershell
cd C:\Users\Administrator\dsv4-local-adapter
$env:UPSTREAM_API_KEY = "sk-你的key"
.venv\Scripts\python.exe -m uvicorn ds_adapter.app:app --host 127.0.0.1 --port 9468
```

## 开机自启

将 `C:\Users\Administrator\dsv4-local-adapter\启动中转工具.vbs` 放入启动文件夹：
`Win+R` → `shell:startup` → 粘贴快捷方式

## 模型切换

- `deepseek-v4-pro`：强推理，适合复杂编程任务
- `deepseek-v4-flash`：轻量快速，适合简单任务

在 GUI 下拉框切换后点保存即可，无需重启适配器。

## 故障排查

| 问题 | 解决 |
|------|------|
| 适配器启动失败 | 检查 Python 依赖：`.venv\Scripts\python.exe -m pip install -r requirements.txt` |
| 上游测试不通 | 确认 API Key 正确且 DeepSeek 账户余额充足 |
| Codex 仍走 OpenAI | 检查 cc-switch 中 Codex 供应商是否已切换为本地中转 |
| cc-switch 导入无反应 | 确认 cc-switch 桌面应用在运行（系统托盘） |
